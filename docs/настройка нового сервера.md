Понял так: тебе нужно **с нуля подготовить Ubuntu-сервер на Timeweb** под запуск `webhook_test.py`, и сейчас только этап подготовки (без сложного прод-настроя). После этого продолжим следующими шагами.

Перед командами важное предупреждение по влиянию:
- **`apt upgrade`** обновит системные пакеты (риск: **средний**, обычно безопасно, но иногда требует перезагрузку).
- Создадим отдельную папку и виртуальное окружение для Python (риск: **низкий**).
- Откроем порт `8000` для теста (риск: **средний**, это временно и только для проверки).

## Шаги для новичка (копируй по порядку)

### 1) Подключись к серверу (на своем ПК)
```bash
ssh root@IP_ТВОЕГО_СЕРВЕРА
```

---

### 2) Базовая подготовка Ubuntu (на сервере)
```bash
apt update && apt upgrade -y
apt install -y python3 python3-venv python3-pip curl ufw
```

Проверка:
```bash
python3 --version
pip3 --version
```

---

### 3) Создай папку проекта (на сервере)
```bash
mkdir -p /opt/webhook_test
cd /opt/webhook_test
```

---

### 4) Загрузи `webhook_test.py` на сервер (на своем ПК, НЕ на сервере)

Открой **новый локальный терминал** и выполни:
```bash
scp "d:\My_dev_project\WR_wh_leads\webhook_test.py" root@IP_ТВОЕГО_СЕРВЕРА:/opt/webhook_test/
```

Если `scp` не работает в PowerShell, используй WinSCP/Termius файл-менеджер и закинь файл в `/opt/webhook_test/`.

---

### 5) Создай `.env` с секретом (на сервере)
```bash
cd /opt/webhook_test
nano .env
```

Вставь строку:
```env
WEBHOOK_SECRET=придумай_длинный_случайный_секрет
```

Сохранить в nano: `Ctrl+O`, Enter, `Ctrl+X`.

---

### 6) Создай виртуальное окружение и поставь зависимости (на сервере)
```bash
cd /opt/webhook_test
python3 -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
pip install fastapi uvicorn python-dotenv python-multipart
```

> `python-multipart` нужен для обработки form-data в твоем скрипте.

---

### 7) Тестовый запуск вручную (на сервере)
```bash
cd /opt/webhook_test
source .venv/bin/activate
uvicorn webhook_test:app --host 0.0.0.0 --port 8000
```

Если видишь внизу что-то вроде `Uvicorn running on http://0.0.0.0:8000` — отлично.

Остановить: `Ctrl + C`.

---

### 8) Открой порт 8000 в firewall (на сервере)
```bash
ufw allow OpenSSH
ufw allow 8000/tcp
ufw --force enable
ufw status
```

---

### 9) Сделай автозапуск через systemd (на сервере)
```bash
nano /etc/systemd/system/webhook_test.service
```

Вставь:
```ini
[Unit]
Description=Webhook Test FastAPI
After=network.target

[Service]
User=root
WorkingDirectory=/opt/webhook_test
EnvironmentFile=/opt/webhook_test/.env
ExecStart=/opt/webhook_test/.venv/bin/uvicorn webhook_test:app --host 0.0.0.0 --port 8000
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
```

Дальше:
```bash
systemctl daemon-reload
systemctl enable webhook_test
systemctl start webhook_test
systemctl status webhook_test --no-pager
```

Логи:
```bash
journalctl -u webhook_test -f
```

---

### 10) Проверка webhook (с сервера или с ПК)

Подставь свой секрет из `.env`:
```bash
curl -X POST "http://IP_ТВОЕГО_СЕРВЕРА:8000/api/provider-test/ТВОЙ_SECRET" \
  -H "Content-Type: application/json" \
  -d '{"test":"ok"}'
```

Ожидаемый ответ:
```json
{"ok":true}
```

---

Когда сделаешь эти шаги, напиши «готово» — дальше помогу:
1) закрыть прямой доступ к `8000`,  
2) поставить Nginx + HTTPS,  
3) подключить домен (если нужен).
