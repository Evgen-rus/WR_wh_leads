# Настройка нового сервера Ubuntu (через `git clone`)

Эта инструкция под твой текущий сценарий: проект уже в Git, на сервере будем делать `git clone`.

Перед началом важно:
- `apt upgrade` обновляет системные пакеты (риск: **средний**; иногда после этого нужна перезагрузка).
- Откроем порт `8000` для теста webhook (риск: **средний**; позже закроем и поставим Nginx + HTTPS).
- Секреты (`.env`) не коммить в репозиторий (риск утечки: **высокий**).

## 1) Подключение к серверу

На своем ПК:
```bash
ssh root@45.153.68.72
```

---

## 2) Подготовка Ubuntu

На сервере:
```bash
apt update && apt upgrade -y
apt install -y git python3 python3-venv python3-pip curl ufw
```

Проверка:
```bash
git --version
python3 --version
pip3 --version
```

---

## 3) Клонирование проекта из Git

Создай папку под проекты и перейди в нее:
```bash
mkdir -p /opt
cd /opt
```

Клонируй репозиторий (подставь только свой GitHub username):
```bash
git clone https://github.com/<USERNAME>/WR_wh_leads.git
```

Перейди в папку проекта:
```bash
cd /opt/WR_wh_leads
```

Проверь, что файл на месте:
```bash
ls
```

---

## 4) Создание `.env` на сервере

В корне проекта:
```bash
nano .env
```

Минимум для `webhook_test.py`:
```env
WEBHOOK_SECRET=придумай_длинный_случайный_секрет
```

Сохранение в nano: `Ctrl+O`, `Enter`, `Ctrl+X`.

---

## 5) Виртуальное окружение и зависимости

```bash
cd /opt/WR_wh_leads
python3 -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
pip install -r requirements.txt
pip install python-multipart
```

`python-multipart` нужен для `form-data` в `webhook_test.py`.

---

## 6) Тестовый запуск вручную

```bash
cd /opt/WR_wh_leads
source .venv/bin/activate
uvicorn webhook_test:app --host 0.0.0.0 --port 8000
```

Если видишь `Uvicorn running on http://0.0.0.0:8000`, все ок.  
Остановить: `Ctrl + C`.

---

## 7) Открыть порт в firewall (для теста)

```bash
ufw allow OpenSSH
ufw allow 8000/tcp
ufw --force enable
ufw status
```

---

## 8) Автозапуск через systemd

Создай сервис:
```bash
nano /etc/systemd/system/webhook_test.service
```

Вставь:
```ini
[Unit]
Description=Webhook Test FastAPI
After=network.target

[Service]
User=root
WorkingDirectory=/opt/WR_wh_leads
EnvironmentFile=/opt/WR_wh_leads/.env
ExecStart=/opt/WR_wh_leads/.venv/bin/uvicorn webhook_test:app --host 0.0.0.0 --port 8000
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
```

Применить и запустить:
```bash
systemctl daemon-reload
systemctl enable webhook_test
systemctl start webhook_test
systemctl status webhook_test --no-pager
```

Логи:
```bash
journalctl -u webhook_test -f
```

## 11) Что указать у поставщика в поле "API ссылка"

В это поле нужно вставить полный URL webhook из твоего `webhook_test.py`:
`/api/provider-test/{WEBHOOK_SECRET}`.

Пока сервер работает напрямую через `uvicorn` на порту `8000`, укажи один из вариантов:

По домену:
```text
http://wrmb-wh.webtm.ru:8000/api/provider-test/ТВОЙ_WEBHOOK_SECRET
```

По IP:
```text
http://45.153.68.72:8000/api/provider-test/ТВОЙ_WEBHOOK_SECRET
```

`ТВОЙ_WEBHOOK_SECRET` — это значение из файла `.env` на сервере:
```env
WEBHOOK_SECRET=...
```

Важно:
- значение секрета должно совпадать 1 в 1;
- после настройки Nginx + HTTPS лучше сменить ссылку у поставщика на:
```text
https://wrmb-wh.webtm.ru/api/provider-test/ТВОЙ_WEBHOOK_SECRET
```
(без `:8000`).

## 12) Что сделать прямо сейчас: оставить `http`, но закрыть внешний `8000`

Этот вариант не дает полной безопасности (без TLS данные идут по сети открыто),  
но заметно лучше текущего состояния: наружу будет торчать только Nginx на `80`.

Риск после настройки: **средний** (из-за `http`), но ниже, чем при прямом доступе к `uvicorn:8000`.

### 12.1 Остановить ручной `uvicorn` (если запущен в терминале)

В терминале, где запущен сервер, нажми `Ctrl + C`.

### 12.2 Поставить Nginx и создать systemd-сервис для `webhook_test.py`

```bash
apt update
apt install -y nginx

cat >/etc/systemd/system/webhook_test.service <<'EOF'
[Unit]
Description=Webhook Test FastAPI
After=network.target

[Service]
User=root
WorkingDirectory=/opt/WR_wh_leads
EnvironmentFile=/opt/WR_wh_leads/.env
ExecStart=/opt/WR_wh_leads/venv/bin/uvicorn webhook_test:app --host 127.0.0.1 --port 8000
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
EOF

systemctl daemon-reload
systemctl enable webhook_test
systemctl restart webhook_test
systemctl status webhook_test --no-pager
```

### 12.3 Настроить Nginx как reverse proxy на `127.0.0.1:8000`

```bash
cat >/etc/nginx/sites-available/wrmb-wh.webtm.ru <<'EOF'
server {
    listen 80;
    server_name wrmb-wh.webtm.ru;

    location /api/provider-test/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
EOF

rm -f /etc/nginx/sites-enabled/default
ln -sf /etc/nginx/sites-available/wrmb-wh.webtm.ru /etc/nginx/sites-enabled/wrmb-wh.webtm.ru
nginx -t
systemctl enable nginx
systemctl restart nginx
systemctl status nginx --no-pager
```

### 12.4 Закрыть внешний `8000`, оставить только SSH и HTTP

```bash
ufw allow OpenSSH
ufw allow 80/tcp
ufw deny 8000/tcp
ufw --force enable
ufw status
```

### 12.5 Проверка

```bash
source /opt/WR_wh_leads/venv/bin/activate
echo "WEBHOOK_SECRET=$WEBHOOK_SECRET"
curl -X POST "http://wrmb-wh.webtm.ru/api/provider-test/ТВОЙ_WEBHOOK_SECRET" \
  -H "Content-Type: application/json" \
  -d '{"test":"ok"}'
```

Ожидаемый ответ:
```json
{"ok":true}
```

Используй адрес без порта `:8000`:

```text
http://wrmb-wh.webtm.ru/api/provider-test/ТВОЙ_WEBHOOK_SECRET
```

Где `ТВОЙ_WEBHOOK_SECRET` — точное значение из `/opt/WR_wh_leads/.env`:

